# ISO 27001 ISMS 導入文件生成系統 - 系統規格書 (System Specification)

## 1. 系統概述
本系統旨在協助無程式開發背景的使用者，透過簡單的問答與表單填寫，自動生成符合 ISO 27001 標準之 ISMS (資訊安全管理系統) 導入與驗證申請所需之各式文件。

## 2. 目標使用者
* 主要使用者：企業內部負責 ISO 27001 導入的承辦人員或主管。
* 使用者特徵：具備基本的公司營運概念，但不具備程式開發或過於深入的資安法規咬文嚼字能力。

## 3. 系統核心功能
1. **四階段導引問卷模組 (4-Stage Guided Questionnaire)**：
   * 系統將 ISO 27001 導入流程拆解為四個結構化訪談階段，引導使用者循序漸進完成：
     * **階段一：專案啟動與全景建構** (收集基本資料與決定驗證範圍)。
     * **階段二：資產盤點與風險評鑑** (引導填寫核心系統與保護措施)。
     * **階段三：管理程序與規定檢核** (確認各項日常管理現況，如密碼、備份)。
     * **階段四：體系鉤稽與最終封裝** (檢查關聯缺漏並打包匯出)。
   * **以範本為基礎的漸進生成 (Template-Driven Approach)**：設計問卷時，需**完全依據各個文件範本的內容結構來回推問題**。確保每一個問卷的題目與選項，都能精確對應到範本中的特定章節、段落或變數，達到「答完題即寫完文件」的目標。
2. **範本引擎與管理模組 (Template Engine & Management)**：
   * 讀取事先建立的標準 ISO 27001 文件範本。
   * **四階層文件架構**：依據 ISMS 規範，將文件劃分為一階 (手冊/政策)、二階 (程序書)、三階 (作業指導書/規範)、四階 (表單/紀錄)，並建立對應的資料夾結構與檔案相依性。
   * **範本提示功能**：針對特定表單提供填寫範例或用途說明（例如：在填寫「內外部溝通管制表」時，系統會提供該表單的用途與範本內容供參考）。
3. **編碼原則與自動命名模組 (Naming Convention Module)**：
   * 內建符合 ISO 27001 要求的編碼設定功能（例如：`IS-001` 代表資訊安全政策）。
   * 系統產出文件時，依據設定的編碼原則自動生成並關聯主檔名與附屬表單檔名。
4. **檢核與鉤稽提醒模組 (Verification & Cross-Check)**：
   * **檢核條件提醒**：當管理程序中提及特定管制條件（例如：密碼長度與複雜度規則、備份週期、定期審查作業等），系統在生成或填寫時主動跳出提醒，要求使用者確認是否符合現況。
   * **文件鉤稽提醒**：設計文件間的關聯檢查機制，確保 A 文件中提及的規定或表單，在 B 文件或產出清單中確實存在，避免文件體系出現斷層。
5. **動態內容調整與差異比對模組 (Dynamic Content Adjustment & Diff Verification)**：
   * **依據產業/屬性動態微調**：根據使用者在問卷中回覆的產業類別或公司特定屬性，系統會自動在文件範本中增刪或修改特定條文與敘述。
   * **變更審閱與確認**：系統在最終產出文件前，會將「預設範本內容 (調整前)」與「客製化內容 (調整後)」以差異比對 (Diff) 的方式呈現給使用者審閱，待使用者確認無誤後才寫入最終文件。
6. **風險評鑑自動統計與分析模組 (Risk Assessment Analytics)**：
   * **自動掃描與過濾**：系統自動讀取並掃描「資產及風險評鑑表」，篩選出所有風險值大於「可接受風險值」的高風險項目。
   * **數據統計與分類**：自動依據資產類別（如硬體、軟體、人員等）對高風險項目進行分類計算（例如：高風險項目共 10 項，軟體類佔 70%）。
   * **分析文字/圖表生成**：將統計結果動態轉換為敘述性文字或圖表，並自動寫入至「風險評鑑報告」中（例如自動填入：「經本次評鑑，高風險資產共計 10 項，其中以 軟體系統類 (70%) 所佔比例最高...」）。
7. **文件生成與匯出模組 (Document Generator)**：
   * 支援生成 `.docx` (Word) 與 `.xlsx` (Excel) 格式之文件。
   * 將生成後的文件依據四階層架構打包妥善，供使用者下載或另存。
   * **落地導讀手冊 (ReadMe) 生成**：隨產出文件附上一份客製化的指引說明，引導使用者後續之簽核發布與填寫作業。

## 4. 系統架構與技術選型 (技術架構)
* **後端核心設計**：`Python`
  * 運用 `python-docx` 處理 Word 範本變數替換與關聯報告生成。
  * 運用 `openpyxl` 或 `pandas` 處理 Excel 表格生成、數據掃描與風險統計運算。
* **網頁後端框架**：`Flask` 或 `FastAPI` (建立輕量化本地或網頁伺服器)
* **前端使用者介面**：`HTML` + `CSS` + `JavaScript` (可能搭配 Bootstrap 等框架以提供直觀美觀的操作介面)
* **資料儲存**：初期以 `JSON` 檔案儲存使用者的問卷回答與系統設定（含編碼規則）即可；未來若需紀錄多版本文件關聯性，可評估導入 SQLite。

## 5. 輸入與輸出設計
* **輸入 (Input)**：使用者透過網頁表單填寫的文字、選項、清單資料 (以 JSON 格式從前端傳送到後端)。
* **輸出 (Output)**：已填入公司客製化資料的 ISO 27001 實體文件檔 (`.docx`, `.xlsx`)。

## 6. 資安與隱私
* 由於涉及公司機敏資訊（資產清單、風險評估），系統設計以**單機執行 (Localhost)** 為主，確保資料不外流至雲端，符合資安精神。
