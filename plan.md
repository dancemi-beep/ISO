# ISO 27001 ISMS 導入文件生成系統 - 開發計畫 (Development Plan)

## 專案開發里程碑 (Milestones)

### Phase 1: 基礎環境準備與概念驗證 ✅
* **成果**: 建立基礎專案結構，確認 Python-docx 與 Openpyxl 的可行性。

### Phase 2: 系統核心生成邏輯 ✅
* **成果**: 實作 `DocumentGenerator` 基礎類別，支援文字替換。

### Phase 3: 四階段導引之前端介面開發 ✅
* **成果**: 使用 Flask + TailwindCSS 建立引導式問卷。

### Phase 4: 系統整合、鉤稽檢查與最終封裝 ✅
* **成果**: 實作 `integrity_checker.py` 與 ZIP 自動壓縮下載功能。

### Phase 5: 使用者手冊與專案交付 ✅
* **成果**: 產出 `README.md` 與安裝啟動腳本。

---

### Phase 6: 範本驅動問卷重構與標記強化 ✅
* **目標**: 讓問卷問題 100% 對應範本內容，達成「答完題即寫完文件」。
* **技術細節**:
  * **批次標籤化 (`batch_tagger.py`)**: 修正 OOOO/XXX 替換規則，確保 59 份範本全自動化處理。
  * **表單提示 (`template_marker.py`)**: 對 15 份操作型表單注入「⚠ 使用者須知」黃底醒目提示。
  * **動態問卷 (`questionnaire_config.json`)**: 依據範本需求逆向推導問卷題目，並實作 Jinja2 動態渲染表單。

### Phase 6b: P2/P3 深層問卷產出機制與動態表單實作 ✅
* **目標**: 實裝高階 ISO 27001 輔導問答邏輯，支援動態段落與表格。
* **技術細節**:
  * **條件式段落渲染**: 在 Word 內使用 Jinja2 `{% if %}` 語法，依據問卷決策（如：BYOD 政策）動態選擇條文內容。
  * **Excel 動態預填模組**: 使用 `openpyxl` 根據「核心系統清單」自動逐列寫入 `IS-004-01` 資產及風險評鑑表。
  * **依決策排除檔案**: 若選擇禁止某項政策，系統會自動在打包時移除對應的附表文件（如 BYOD 申請單）。

### Phase 7: 測試回饋修復與自動化歸檔模組實作 ✅
* **目標**: 修復測試除錯議題，強化系統報表與使用者範例。
* **技術細節**:
  * **重構目錄體系**: 自動依政策/程序生成 `表單/`、`紀錄/` 子目錄，完美還原 ISO 27001 四階層歸檔架構。
  * **樣式強化**: 使用 `python-docx` API 在生成後強制將封面公司名稱設為 24 級粗體。
  * **報表產生器**: 新增 `questionnaire_report.py` 自動產出 `問卷填答總表.docx`。
  * **範例自動掛載**: 打包時自動巡航 `examples/` 目錄並依命名規則分發至對應子目錄。

### Phase 8: 企業形象整合與全局編號同步 ✅
* **目標**: 品牌形象落實與跨文件編號一致化。
* **技術細節**:
  * **企業形象注入**: 導入「諮力管理顧問」主視覺、標語、資深顧問聯絡名片與 100% Localhost 隱私聲明。
  * **全局編號同步**: 
    - **檔名與目錄**: IS 變更為自訂前綴（如 ABC）。
    - **內容掃描器**: 產出後自動遍歷 Word (段落/表格/頁首頁尾) 與 Excel (儲存格)，將內容中所有 `IS-` 參考編號同步替換為自訂前綴。
  * **ISMS 範圍動態化**: 將 `IS-001` 資訊安全政策之範圍描述改由問卷欄位 `isms_scope` 動態產生。
