# ISO 27001 ISMS 文件生成系統 — 安裝與使用手冊

> 本手冊適用對象：拿到系統封裝檔的終端使用者（ISO 27001 導入承辦人員）。
> 無需任何程式背景即可操作。

---

## 一、您收到的檔案

封裝檔解壓後，資料夾結構如下：

```
iso_doc_generator/
├── start.sh                 ← 一鍵啟動腳本（雙擊或終端機執行）
├── requirements.txt         ← 系統所需套件清單（自動安裝）
├── README.md                ← 開發者參考文件
├── document_structure.json  ← 文件結構定義（勿修改）
├── files/
│   └── marked/              ← 59 份 ISO 27001 範本（勿刪除）
└── src/                     ← 系統程式碼（勿修改）
```

> ⚠️ **請勿刪除或重新命名上述任何檔案或資料夾**，否則系統將無法正常運作。

## 二、安裝前置作業（僅首次需要）

### 2.1 安裝 Python

本系統需要 **Python 3.10 以上**版本。請依據您的作業系統執行以下步驟：

#### macOS

1. 開啟「**終端機**」（按 `Cmd + 空白鍵`，輸入「終端機」或「Terminal」）
2. 輸入以下指令確認是否已安裝：
   ```
   python3 --version
   ```
3. 若顯示 `Python 3.10` 以上版本，**可直接跳到第三節**。
4. 若未安裝，前往 https://www.python.org/downloads/ 下載安裝。

#### Windows

1. 前往 https://www.python.org/downloads/ 下載最新 Python 安裝檔。
2. 執行安裝程式時，**務必勾選「Add Python to PATH」**。
3. 安裝完成後，開啟「**命令提示字元**」（按 `Win + R`，輸入 `cmd`），輸入：
   ```
   python --version
   ```
4. 確認顯示 `Python 3.10` 以上版本。

---

## 三、啟動系統

### macOS / Linux

1. 開啟「**終端機**」。
2. 進入系統資料夾（將路徑替換為您實際解壓的位置）：
   ```
   cd /您的路徑/iso_doc_generator
   ```
3. 首次執行需賦予啟動腳本權限：
   ```
   chmod +x start.sh
   ```
4. 啟動系統：
   ```
   ./start.sh
   ```

### Windows

1. 開啟「**命令提示字元**」。
2. 進入系統資料夾：
   ```
   cd C:\您的路徑\iso_doc_generator
   ```
3. 建立虛擬環境並安裝套件（首次）：
   ```
   python -m venv venv
   venv\Scripts\activate
   pip install -r requirements.txt
   ```
4. 啟動系統：
   ```
   python src\web\app.py
   ```

### 啟動成功畫面

當您看到類似以下訊息，表示系統已就緒：

```
🚀 啟動伺服器中...
 * Serving Flask app 'app'
 * Running on http://127.0.0.1:5001
```

5. **開啟瀏覽器**（Chrome、Edge、Safari 皆可），在網址列輸入：

   **http://localhost:5001**

## 四、使用流程

系統採用 **四階段引導問卷**，逐步協助您完成所有文件設定。

### 階段一：專案啟動與全景建構

填寫公司的基本資料，包括：
- 公司全名與簡稱
- 文件發布日期與版本號
- 權責部門名稱
- 驗證範圍說明

> 💡 所有欄位皆有預設提示，依實際情況填寫即可。

填寫完畢後，點擊 **「下一步」** 進入階段二。

### 階段二：資產盤點與風險評鑑

設定與風險評鑑相關的參數，包括：
- 可接受風險值（門檻值）
- 資產類別與評估方式

填寫完畢後，點擊 **「下一步」** 進入階段三。

### 階段三：管理程序與規定檢核

確認各項管理程序的現況設定，例如：
- 密碼規則
- 備份週期
- 其他管制措施

填寫完畢後，點擊 **「下一步」** 進入階段四。

### 階段四：體系檢查與一鍵生成

此頁面分為兩個區塊：

#### A. 鉤稽檢查結果面板

系統會自動檢查以下項目並顯示結果：
- ✅ **通過**（綠色）：所有範本齊全、結構正確
- ⚠️ **警告**（黃色）：有可跳過的注意事項
- ❌ **錯誤**（紅色）：必須修正才能繼續

#### B. 一鍵生成按鈕

確認鉤稽無錯誤後，點擊 **「⚡ 一鍵生成全套 ISMS 文件」**。

系統將自動執行：
- 替換 59 份文件中的所有變數
- 計算風險評鑑分數
- 產出導入指引手冊
- 將所有文件壓縮打包

> ⏳ 生成過程約需 10~30 秒，請耐心等候。

## 七、常見問題 (FAQ)

### Q1：出現 403 Forbidden / 存取遭拒

**原因**：macOS 的 AirPlay 接收器佔用了 port 5000。
**解法**：本系統已預設使用 port **5001**，請確認網址為 `http://localhost:5001`。

### Q2：啟動時顯示「找不到 python3」

**解法**：請先安裝 Python 3.10 以上版本（參見第二節）。

### Q3：生成時出現部分檔案「跳過」

**原因**：部分檔案格式為 PDF 或其他無法處理的類型，屬正常情況。
**影響**：不影響核心 ISMS 文件的完整性。

### Q4：想修改已產出的文件

直接用 Word 或 Excel 開啟 `output/` 目錄中的文件進行編輯即可。

### Q5：想重新生成（例如公司名稱填錯）

回到瀏覽器首頁 `http://localhost:5001`，重新填寫問卷並再次生成，舊檔案會被覆蓋。

### Q6：如何停止系統

在終端機中按 **`Ctrl + C`** 即可停止伺服器。
